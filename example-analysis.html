User Manual on Labor Market Information

9 Example Analysis

In this section we present a few examples for VR agencies of how to crosswalk LMI data sources to answer questions of relevance. Our goal is to present examples, and share as much methodology as possible so that these examples can be replicated by VR agencies.

Examples presented in this section covers different sources of LMI. Each example describes the data source, how data was accessed, caveats associated with the data, purpose of the analysis, methodology used and findings. Further we also discuss the utility and how each example can be replicated.

Purpose of this manual is not to only illustrate the utility of LMI, but it is also to contextualize LMI’s utility vis-à-vis VR. This section looks into how LMI can be used to meet employers’ needs, serve clients, improve outreach and engage with different stakeholders.

9.1 Occupational Statistics

Purpose

To answer the question: What are the annual wages for entry level workers and experienced workers in selected occupations in a state.

Data Source

OES Wage Survey, Bureau of Labor Statistics

Occupational Economic Survey (OES) is an annual survey that collects data on occupation trends and wages across the country. The BLS uses the standard occupation classification code (SOC) that is also used by VR agencies. Occupational code data are available and can be linked with RSA 911 Data can be examined at the state, Metropolitan Statistical Area (MSA) level and at the Metropolitan Division and Nonmetropolitan Area.

Accessing the Data

Data is available on the OES page in BLS’ website https://www.bls.gov/oes/home.htm. A visitor can use tools available on the website https://www.bls.gov/data/ and it provides a profile for each occupation within an industry and across locations. Data in tabular format is made available for National, State, Metropolitan and nonmetropolitan area is available at https://www.bls.gov/oes/tables.htm.

Things to think about

OES covers all full-time and part-time wage and salary workers but does not include those that are self-employed, owners and partners in unincorporated firms, household workers or unpaid family workers. OES refers to 6 digit NAICS codes for identifying industries for all the occupations.

Methodology

Following analysis is based on state level OES data available at https://www.bls.gov/oes/tables.htm. We aggregated wage data for different major occupations (OCC_TITLE), and presented following statistics –

  • number of people employed ($) (TOT_EMP),
  • mean annual wage at 25th percentile ($) (A_PCT25),
  • mean annual wage ($) (A_MEAN), and
  • mean annual wage at 75th percentile ($) (A_PCT75).

Utility

Alone, occupation data is useful in appreciating the trends and forecasting. Together with other data sources for example VR’s RSA 911 - which uses SOC to represent successfully closed cases – OES can be used to present more holistic picture of labor market and occupational demand specifically.

Replication The OES is a very accessible dataset. The BLS provides aggregated data tables for easy analysis. The data tool made available at https://www.bls.gov/data/ can be used to find very specific occupational statistics. The BLS also provides Application Programming Interface (API) for programmers and developers to develop tools for more robust analysis. Such tools, can be developed at an institutional level. More information BLS’ API is available at https://www.bls.gov/developers/.
